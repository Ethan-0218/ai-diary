## 1. 비전 / 북극성 ✅

> **매일 열어서 친구한테 수다 떨면, 그게 다시 보고 싶은 내 문체의 일기로 남고, 그 친구는 나를 기억한다.**

세 가지 감정을 준다: ① **말 거는 맛**(대화로 빈 페이지 제거) ② **받아보는 맛**(시시하지 않은 결과물) ③ **기억되는 맛**(나를 알아봄).

## 2. 타겟 유저 ✅

이 제품이 푸는 진짜 문제 — **일기는 다들 좋은 줄 알지만 거의 다 실패한다**:
1. **빈 페이지 공포** — 막상 쓰려면 뭘 써야 할지 막막
2. **귀찮음/작심삼일** — 혼자선 동기가 안 붙음
3. **결과물이 시시함** — "오늘 ~했다" 나열이라 다시 안 봄

세 무기가 정확히 이 셋을 때린다: **대화**(빈 페이지 제거) · **친구/기억**(혼자가 아님) · **멀티포맷**(결과물이 작품).

### 1순위 타겟 = A+B: "못 쓰는 기록·정리러" ✅
> 기록·자기이해 욕구는 **있으나**, 혼자선 빈 페이지와 꾸준함 앞에서 무너지는 사람. 하루 끝 감정이 정리되지 못한 채 흘러간다.

- **A. 못 쓰는 기록러** (지우, 29) — 다이어리앱 깔았다 지웠다 반복. 욕구는 큰데 손이 안 감. → *마찰 제거 + 꾸준함*.
- **B. 마음 정리러** (서연, 34) — 하루 끝 감정이 복잡, 털어놓을 데는 없음. → *들어주는 위로 + 정리*.

A를 코어로, **B의 정서를 입힌 합본**. 마찰 제거가 핵심 가치라 **리텐션(매일 씀)**으로 직결된다.

### 일부러 1순위에서 빼는 것 (non-target)
- **C. 표현·창작러** — 멀티포맷의 매력은 타겟이 아니라 **차별화 후킹·재방문 보상**으로 둔다(단독 타겟은 얇음).
- **D. 외로운 동반자** — companion으로 가면 "AI 친구" 카테고리가 되어 일기 정체성이 흐려지고 정서·기대치 리스크. **기억은 "일기를 더 잘 써주기 위해"서만** 쓴다.
- **E. 추억 보관러** — 사진 기능으로 자연히 보조 충족.

## 3. 핵심 경험 (받고 싶은 경험) ✅

**한 줄**: *"오늘 하루를 누군가 다정히 물어봐줬고, 그게 다시 보고 싶은 내 글로 남았다."*

### JTBD (Job To Be Done)
> "하루를 마칠 때, **큰 노력 없이** 오늘을 돌아보고 마음을 정리하고 싶다. 그리고 그게 **나중에 다시 보고 싶은 기록**으로 남으면 좋겠다."

### 한 세션의 감정 아크
1. **진입**: 부담 0. "뭘 써야 하지"가 없다 — 친구가 **먼저** 물어봐 준다.
2. **대화 중**: 들어주는 느낌, 내 말에 진짜 반응. 감정을 **추측-확인**으로 끌어내 줌 → "아 맞아, 그랬지" 정리되는 감각.
3. **받음**: 내 어휘로 쓰인, 시시하지 않은 한 편. "내가 이런 하루를 보냈구나" 뿌듯/위로.
4. **누적**: 얘가 날 기억한다 → 이어지는 느낌, 안 쓰면 아쉬움.

### 가치 우선순위
① **마찰 제로(쉬움)** → ② **정서적 보상(들어줌+정리)** → ③ **결과물 품질(다시 보고 싶음)** → ④ **연속성(기억)**

### 톤 & 페르소나
- 다정한 **반말 친구**. 심문 금지, 매 턴 질문 강제 금지, 진짜 반응. (agent-v2 §2 계승)
- 일기는 **사적·비공개 기본**. 공유는 부차.

### Non-goals (안 만드는 것)
- AI 친구 채팅앱(관계 자체가 목적) ✗
- SNS 공유 피드 플랫폼 ✗
- 감정분석/무드 트래커 대시보드 ✗

## 4. 정보구조 / 화면 맵 ✅(모델·홈) 🚧(상세)

### 핵심 모델 ✅ — "하루 = 하나의 이어지는 대화 → 한 편의 일기"
- 하루 일기는 **한 편** (회고 습관에 집중).
- 그 한 편은 하루에 걸쳐 **이어지는 단일 대화**에서 나온다 — 아침·점심·저녁 **여러 번 돌아와** 잇는다.
- **생성 시점 = 유저가 "일기 만들기"를 누를 때.** AI는 충분하면 *제안*만 하고, 결정권은 유저.
- 생성 후에도 **보강** 가능 → 더 대화하고 다시 생성(최신 1편으로 갱신).
- (메모) **하루 경계** ❓ 자정 vs 새벽 보정 · **과거 날짜 작성**은 후순위.

### 적응형 홈 ✅ — 오늘의 상태에 따라 변함
홈을 "무조건 대화"로 고정하지 않는다. 타겟 A+B의 일상 작업은 "오늘을 정리한다" → 그 작업이 **남았으면 대화가 전면, 끝났으면 결과물이 전면**.

- **상태 1 — 오늘 대화 없음** (습관 루프 핵심): AI 선인사(시각·날씨·기억 한 줄) + `오늘 이야기하기`. `지난 일기 ▸` 보조.
- **상태 2 — 대화 중, 일기 아직**: `이어서 이야기하기` + (충분하면) `일기 만들기` 유도.
- **상태 3 — 오늘 일기 있음**: 오늘 일기 카드가 주인공 + `더 이야기해 보강하기`(강요X). `지난 일기 ▸`.

> 위 3상태는 **일기장 한 권 기준**. 여러 권일 때 홈은 종류로 묶임(결정 B): **firm 존(상단)** = 기간형들의 "오늘"(능동) / **soft 존(하단)** = 칸형들의 진행도+이어쓰기(가볍게).

### 화면 목록 (1차)
1. **홈 (적응형)** — 위 3상태
2. **대화 화면** — 오늘의 이어지는 대화 (포맷 중립)
3. **일기 뷰** — 페이오프, 인라인 사진
4. **일기 수정** — 생성된 글을 직접 손보기 (보강과 구분, §6)
5. **지난 일기 / 보관** — 리스트·캘린더, 다른 포맷으로 다시쓰기
6. **온보딩 / 로그인**
7. **설정 / 계정**
- 기억 = 거의 invisible (별도 화면 후순위)

## 4.5 일기장(notebook) 모델 🚧 — 중심 은유 & 동기·수익

> 발산에서 나온 핵심 아이디어. "구독"이 아니라 **일기장을 산다**. 일기장 = `포맷 × 기간/칸 × 가격`이 정해진 한 권. **다 채우면 한 권의 작품.**

**왜 채택 방향인가**
- A+B의 작심삼일을 *"이 한 권을 채운다"*는 목표로 때림 (종이 일기의 동기 + 앱의 마찰 제로).
- 보상이 "한 편" → **"한 권(책/작품)"** 으로 스케일업 (간직·선물·다시읽기).
- **포맷이 일기장 속성**이 됨 → 매일 대화엔 포맷 마찰 0, 그러면서 대화가 포맷을 알아 **인터뷰 깊이 부활**. (§6 "포맷 선택 시점" 논쟁 해소)

**IA 영향**: 홈 위에 **책장(bookshelf)** — 진행중/완성 일기장. 일기장 탭 → 그 권의 "오늘"(적응형 홈 3상태) 그대로 작동. 일기장은 그릇.
**기억**: 유저 단위(일기장 넘나듦). **친구는 한 명, 일기장만 여러 권.**

### 결정 A ✅ — "기간" 모델 = 두 종류 다 제공, 구매 시 유저 선택
- **기간형 (연대기)** — "2026년 6월의 일기"처럼 달력 단위 고정. 차곡차곡·달력 회고("작년 몇월에"). 빈 날 가능. **더 저렴**(제약·미완 리스크). → *내 삶의 척추.*
- **칸형 (테마/챌린지)** — "30칸 채우기", 날짜 무관, 완성 시 "첫날~마지막날" 자동 라벨. 관대함·완성 보장. **더 비쌈**(유연 프리미엄). → *특별 프로젝트.*
- **가격 원칙**: 제약(기간형) < 유연(칸형).
- 책장은 *연대기 줄 / 컬렉션 줄*로 나눠 보여줄 수 있음.
- **단서 1**: 구매는 조합 매트릭스가 아니라 **큐레이션된 한 권**(「6월의 소설」 등)으로 진열 — 마찰 제로 유지.
- **단서 2**: 기간형 미완 = **비난 없는 언어**("6월 · 8편"), 빠진 날 들이밀지 않음.

### 결정 B ✅ — 여러 권 설계 내장, 홈/알림 강도는 일기장 "종류"가 결정
- 일기장 여러 권 동시 보유·진행 가능 (데이터·책장 N권 **처음부터** 전제. 나중에 끼우면 복잡도↑).
- **알림 규칙 = 종류 따라**:
  - **기간형(연대기) → 능동(firm)**: 미루면 영구 빈 칸 → 홈 상단 + 저녁 알림. **2권+여도 다 띄움.**
  - **칸형(테마) → 수동(soft)**: "오늘 안 씀"을 부담 없이(진행도+이어쓰기 초대만). 일일 졸림 없음.
- 결정 A의 의미(기간형=압박/칸형=관대)와 알림 강도까지 일관.
- "하루 한 편" = **일기장 단위** / 친구·기억 = **유저 단위 공유.**
- 별도 "핀" 불필요 — firm 상단 / soft 하단으로 자연 정렬. (보통 연대기 1 + 칸형 몇 → 차분한 홈)

### 결정 C ✅ — 무료 스타터 일기장 = 기간형 3칸, 제한 포맷 택1
- 신규 유저에게 **무료 스타터 일기장 1권** 지급 → 대화→일기→책장→**완성** 루프 + 완성 동기를 먼저 체험. (사기 전 마법 경험; "온보딩 페이오프" 갈림길도 해결)
- **기간형 3칸(오늘·내일·모레)**, 일부러 firm(매일 알림) → 온보딩에서 **매일 돌아오는 습관·알림**을 체험시켜 리텐션 시동. (미완 언어는 여전히 비난 없이)
- **포맷 = 제한 선택**: `일반` + `임팩트 포맷 1종(예: 소설)` 중 **택1**. 받을 때 고름(모델과 일관). 멀티포맷 USP를 첫인상에 노출.
- **이미지 다용 포맷(그림일기 등)은 무료 체험 제외** — 비용. 유료 일기장에서만.
- 다 안 써도 구매 가능(막지 않음). 스타터 맛본 직후 스토어로 자연 유도.

### 결정 (음성 입력 STT) ✅
- **음성 답변(STT)** 추가 — 타이핑 부담 제거 = **#1 가치(마찰 제로) 직격**. "친구한테 말하듯"이 진짜 말이 됨(강한 차별화 후킹).
- **과금 = 음성도 일기장 속성** (포맷·길이처럼). "음성 일기장" 구매 또는 기존 일기장 **업그레이드**로 켬.
- **미터 없음** — 일기장 칸 수가 유한 → 음성/LLM 변동비도 유한 → 일기장 가격에 녹임. **그 일기장 내 음성 무제한.** (미터 불안 = 마찰이라 배제)
- **텍스트는 항상 가능**, 음성은 옵션·혼용. 강제 아님.
- **무료 스타터**: 음성 미포함이나 **마지막(3번째) 칸에서 음성 1회 맛보기** 활성 → 2번 타이핑으로 마찰 느낀 뒤 마지막에 음성 임팩트 → 완성+업셀 극대화.
- ※ 일기장 모델이 **완성 동기 + 변동비(STT·LLM) 상한**을 동시에 묶어줌.

### 결정 F ✅ — 소유(buy-to-own) 중심 + 번들로 장기 약정
- **일기장을 산다(소유).** 구독은 소유·완성 감성("결제 끊기면 사라지나?" 불안)을 깎아 **1차 제외.**
- **번들로 구독 역할 대체** — 기간형 **12권 묶음(연간)** = 1년 약정·할인을 소유 모델 안에서. 책장이 12칸 연대기로 차오르는 **거시 완성 목표**까지 생김.
- 구독은 후순위 옵션으로만 비전 메모(헤비 유저 무제한/할인 패스).
- IAP 상품 타입(소비성/비소비성)·가격대는 **S4(IAP 셋업)로 위임.**

> ✅ **§4.5 일기장 모델 확정** (A 기간/칸 · B 여러권·알림 · C 스타터 · STT · F 소유·번들). 그림일기(D)는 §6에서 비전 포함·출시 후순위.

## 5. 핵심 플로우 명세 🚧

> S3.1 끝 기준. 아래 플로우를 각각 단계별로 명세한다.

### 5.1 온보딩 (첫 실행 → 첫 일기 → 스타터 완성 → 업셀) ✅
1. **가치 소개** — 로그인 전, 2~3장 스와이프(스킵 가능). 샘플 소설일기로 페이오프 암시.
2. **무료 스타터 받기 + 포맷 택1** — "무료 3일 일기장 받기" → `일반`/`소설` 중 택1.
3. **로그인/회원가입** — *받기를 누르면* 트리거(가입 동기 또렷). Apple/Google/Kakao 원탭 + 약관·개인정보 동의.
4. **일기장 수령 모션** — 가입 완료 = **일기장이 책장에 꽂히는 모션**(보상·소유감). "오늘부터 같이 채우자."
5. **Day 1 대화** — 홈 `오늘 이야기하기` 탭 → **이때 위치 권한 요청**("날씨도 적어줄게, 위치 켤래?") → AI가 **시각+날씨** 얹어 선인사(첫날=기억 없음) → 수다(텍스트) → "오늘 일기로 남겨줄까?" → `일기 만들기`. (위치 거부 시 날씨 없이 인사)
6. **첫 일기 받기 ★매직** — 생성 모션 → 일기 등장 → "네 첫 장, 책장에 꽂아뒀어. 내일 또 보자." → **알림 권한 요청**("내일 이 시간에 부를까?").
7. **Day 2 / Day 3** — 알림 복귀, 같은 루프. **Day 3(마지막 칸) 음성 맛보기** 활성("이번엔 말로 해볼래?").
8. **스타터 완성 → 업셀** — 완성된 작은 책 + "이제 진짜 네 일기장 만들래?" → 스토어(음성 일기장 자연 유도).

> **권한 타이밍 원칙**: 필요한 맥락에서. 위치 = **대화 시작**(인사에 날씨 쓰려면 그 전) · 알림 = **첫 일기 직후**(복귀 동기).

### 5.2 일상 핵심 루프 ✅
1. **홈 진입**(앱/알림) → 적응형 홈(상태1/2/3, firm·soft).
2. **선인사** — 시각·날씨 기반.
   - **기억 훅 = "예정 기억"만**: 유저가 전에 *"○○일에 ~할 거야"* 한 게 **오늘이면** 그걸 물음("오늘 면접 날이지, 어땠어?"). → 친구가 내가 중요하다 한 걸 기억해준 느낌.
   - **금지**: 일반적 "어제 ~어땠어?"식 회상으로 시작 ✗(심문·억지 연속성). 일반 기억은 인사로 안 꺼내고 대화·일기 품질에 *은근히*. (§5.6)
3. **대화 진행** — 텍스트/음성 응답 → 진짜 반응 + 추측-확인 감상 → 사진 제안/첨부 → 하루 여러 번 재진입(이어서, 선인사는 첫 진입만).
4. **일기 만들기 (가)** — 버튼 **상시 접근**(유저가 언제든 끊고 생성) + AI가 충분하면 능동 제안. 강요 없음.
5. **생성 — 충실도 가드(길이=입력 비례)**:
   - **일반**: 입력 적으면 담백하게 **짧게**. 억지 패딩·감상 창작 금지(§2-1 우선).
   - **소설**: 문학적 **윤색으로 확장 허용**(없는 사건·사실 날조는 금지).
   - **신문**: 사실·육하원칙, 과장 금지.
6. **일기 뷰** — 포맷 렌더(인라인 사진). 그날 칸 채워짐(+1). 액션 `보강`/`고치기`/닫기(→홈 상태3).
7. **보강 / 수정** — 보강=대화 더→재생성(최신 1편, **칸 수 불변**, 그날 새벽5시 컷까지〔잠정〕) / 수정=AI 지시로 글 다듬기. 횟수 캡=결제단계.

### 5.3 스토어·구매 🚧 잠정 (구현 시 상세화)
- **진입점**: 스타터 완성 업셀 / 책장 `+ 새 일기장` / 기간형 만료 임박 / 음성 업그레이드.
- **스토어 구조**(책장과 미러): **연대기 섹션**(기간형 월·연·12권 번들) / **컬렉션 섹션**(칸형 테마·챌린지). 상품 = **큐레이션 SKU 카드**(표지·이름·포맷·기간/칸·음성·가격), 조합 커스텀 ✗.
- **구매**: 카드 → 상세 → IAP → 책장 꽂힘 모션. 같은 종류 여러 권 가능(기간형은 기간당 1권).
- **음성 업그레이드**: 기존 일기장에 `음성 켜기` IAP → 남은 칸 음성.
- **기간형 만료·전환**: 달 끝(현지 새벽5시) 자동 마감("N편") → 다음 권 있으면 자동 활성 / 없으면 홈에서 구매 유도. 미래 기간 예약 구매 허용.
- ※ 가격대·IAP 상품 타입·예약 배치 세부는 **구현/S4에서 상세화.**

### 5.4 책장·회고 ✅ (잠정 디테일)
- **책장**: 진행중/완성, **연대기 줄 / 컬렉션 줄**, 표지+진행도. 일기장 열기 → 칸 목록(기간형=캘린더, 칸형=그리드) → 개별 일기 뷰.
- **회고**: 연대기에서 과거 월 → 그날. 검색(키워드/의미)은 후순위.
- **다른 포맷 다시쓰기 = 없음 ✅** — 포맷은 *구매한 일기장*으로 정해짐 → 다른 포맷은 그 포맷 일기장에 그날 쓰면 됨(결정 B). 일회성 변환(b)은 미채택.

### 5.5 사진·날씨 🚧 잠정 (PoC 자산)
작성칸 동반 전송 + AI vision 캡션 + 본문 인라인 배치 + HEIC→JPEG + 날씨(Open-Meteo+역지오코딩, 위치권한 §5.1). **멀티 사진/순서는 후순위**.

### 5.6 기억 (대부분 invisible) 🚧 잠정
인사 훅 = 예정 기억(§5.2). 일반 기억은 대화·일기에 *은근히*. **기억 열람·수정·삭제 UI는 후순위**(출시 후). 추출은 보수적(명시 사실만).

### 5.7 설정·계정·알림 🚧 잠정
**계정 삭제**(스토어 필수) · 구매 복원 · 로그아웃 · **알림 시간 설정**(저녁 리마인더 기본) · 개인정보/약관.
