# 플랜 — 데모 → 출시 가능한 완성품(1.0) (마일스톤 1)

> **방식**: 큰 단계만 먼저 정한다. 각 단계 상세는 **그 단계에 도착했을 때** JIT로 따로(작게) 짠다. — `plan-to-testflight.md`와 동일.
> **목표(이 마일스톤)**: 마일스톤 0에서 **파이프라인(앱→IAP→배포→TestFlight 검수 제출)** 을 끝까지 뚫었다. 이제 그 위에 **S3.1 기획의 풍부한 경험 + 생성 품질 + 운영 견고함**을 올려 **실제로 매일 쓰고 싶고 출시할 수 있는 1.0**으로 완성한다.
> **북극성**: *매일 열어서 친구한테 수다 떨면, 그게 다시 보고 싶은 내 문체의 일기로 남고, 그 친구는 나를 기억한다.* (`docs/s3.1-service-ux.md` §1)
> **타겟·가치순위**: A+B "못 쓰는 기록·정리러" / **쉬움(마찰제로) > 정서보상 > 결과물(품질) > 기억.** 우선순위는 이 순서를 따른다.

## 현재 상태 (마일스톤 0 종료 시점, 2026-06-09)

**된 것**
- 백엔드(NestJS+Postgres+pgvector) **EC2 배포** + HTTPS(`aidiary-api.samshintalk.com`). 노트북 소유 모델·상품·결제 검증·기억(M3).
- 모바일 **코어 루프 이식**: Login(Apple) → 책장(노트북 목록) → Chat(스트리밍) → Diary(마크다운·보강·수정) + Store(IAP 구매·검증) + 무료 스타터.
- TestFlight build 3 외부 검수 제출(Waiting for Review). 타임존 버그 수정.

**아직 없는 것 (= 이 마일스톤에서 채움)**
- **온보딩** 전무(로그인 후 바로 책장).
- 홈이 **적응형(3상태)** 아님 — 단순 목록. **책장**도 평면 리스트(캘린더/그리드·표지·진행도·칸 목록·회고 없음).
- **알림**(리마인더·예정기억 훅) 없음. **음성 입력(STT)** 없음.
- 상품 **일부만**(W4 4종) 등록·판매. 나머지 티어·번들·음성 상품·큐레이션 UX 미완. 주간 비례 가격 표시 UX 미완.
- 앱 **이름**이 `AiDiary`(→`AI 일기`), 앱 **아이콘** 플레이스홀더.
- 운영 debt: prod **마이그레이션 미전환**(synchronize=true), 환불 웹훅 없음, 모니터링 미비.

## 큰 단계 (이것만 먼저 확정, 상세는 JIT)

| 단계 | 큰 그림 | "끝" 기준 |
|---|---|---|
| **P0. 마감(첫인상)** | 앱 이름 `AiDiary`→`AI 일기` · 앱 아이콘 디자인 교체 | 홈화면 이름·아이콘이 "완성품"처럼 보임 |
| **P1. 핵심 루프 완성** | **적응형 홈**(오늘 상태 3 · firm/soft) + **제대로 된 책장**(연대기/컬렉션 줄·표지·진행도·칸 목록=캘린더/그리드·회고) + 일기 뷰/보강/수정 다듬기 | 매일 쓰는 그릇이 §4 기획대로 동작 |
| **P2. 온보딩 + 첫 일기 매직** | §5.1: 가치소개 → 스타터 받기(+로그인) → 일기장 수령 모션 → Day1 대화(위치권한) → 첫 일기 페이오프(알림권한) → 스타터 완성·업셀 | 신규 유저가 첫 일기까지 매끄럽게 도달 |
| **P3. 생성 품질 튜닝** | 실사용 기반 페르소나·인터뷰(상태머신)·가드·문체/길이 개선 + 간단 평가 루프 | "다시 보고 싶은 내 글" 체감 향상 |
| **P4. 알림·리텐션** | 저녁 리마인더(기간형 firm/칸형 soft) + 예정 기억 greeting 훅(§5.2) + 권한 타이밍 | 알림으로 복귀하는 습관 루프 작동 |
| **P5. 음성 입력(STT)** | 말로 일기 — 음성 응답(STT) + 음성=일기장 속성·과금(미터 없음, 권 내 무제한) | 음성으로 대화→일기 완성 |
| **P6. 커머스 마감** | 나머지 티어(W3~W1)·**12권 번들**·**음성 업그레이드** 상품 + 스토어 큐레이션 카드·섹션 + 주간 비례 가격 표시 | 전체 상품 라인업이 진열·판매·발행 |
| **P7. 인프라·품질 하드닝** | prod **마이그레이션 전환** · **환불 웹훅**(App Store Server Notifications V2) · 모니터링/로깅 · 에러 하드닝 · App Privacy(스토어 영양표) · (선택) Google/Kakao 로그인 · 정식 도메인 | 운영 가능한 견고함 |
| **P8. 정식 출시** | App Store 스크린샷·설명·키워드·IAP 스크린샷·가격 최종 → App Review 제출 → 승인 | App Store 정식 출시(✅ 마일스톤 완료) |

> **권장 진행 순서**: P0(워밍업) → **P1·P2·P3**(제품의 심장: 첫인상~핵심경험~결과물) → P4·P5(리텐션·차별화) → P6·P7 → P8. 단, 순서는 JIT로 조정 가능.
