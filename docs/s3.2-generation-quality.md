## 1. ⚠️ 일기장 모델이 agent-v2 인터뷰 설계를 다시 흔드는 3가지

agent-v2 §4-B의 인터뷰 설계는 **"한 번 앉아서 하는 한 세션"**을 전제로 짜였다. S3.1 일기장 모델이 이 전제를 3군데서 깬다 — 이번 기획의 핵심 제약:

1. **세션 ≠ 한 번 앉기. 하루에 걸친 단일 대화 + 여러 재진입** (s3.1 §4). → `greeting→gather→enough→offer` 상태가 **하루 단위로 누적**돼야 한다. greeting(예정기억 훅)은 첫 진입만, 재진입은 "이어서". gather 상태는 아침·점심·저녁에 걸쳐 쌓인다. → **A에서 풀 숙제.**
2. **"enough?"가 게이트가 아니다.** 일기 만들기 버튼이 **상시 접근**(s3.1 §5.2-4)이라 enough 신호는 *"이제 일기 제안해도 될까"* 하는 **부드러운 제안 트리거**일 뿐. 조기종료 방지의 의미가 "강제로 더 묻기"가 아니라 "충분해질 때까지 *능동 제안을 미루기*"로 바뀐다. → **A에서 풀 숙제.**
3. **포맷이 대화 시작부터 확정**(일기장 속성). → agent-v2가 "포맷 선택 시점" 때문에 못 살렸던 **인터뷰 깊이가 부활**. 1턴부터 포맷-aware. **단 이게 페르소나 충돌을 만듦 → §2에서 해소.**

추가 변수(agent-v2엔 없던 것): **음성(STT) 입력**(더 길고 거친 원재료), **보강**(생성 후 재대화→재생성으로 파이프라인이 하루 안에서 루프). → C/D에서 닿음.

## 2. 페르소나 = 일기장 속성 ✅ 확정

> **결론**: 페르소나는 `포맷·기간·음성·가격`처럼 **일기장의 한 속성**이다. 일기장마다 **인격이 다르되, 기억은 유저 단위로 하나**.

### 충돌과 해소
- **충돌**: s3.1 §4.5 "친구는 한 명"(마찰 제로) ↔ formats.ts "기자/소설가" 페르소나. "신문 일기장 사면 친구가 매일 기자로 심문하나?"
- **해소 1 — "친구는 한 명"의 재해석**: 그 문장의 의도는 *"기억·관계가 하나(일기장 넘나듦)"*지 *"목소리가 하나"*가 아니다. → 기억 공유 + 인격은 일기장별, 양립.
- **해소 2 — 마찰 걱정은 틀렸다**: 유저가 **신문 일기장을 일부러 골랐으면** 리포터 다이내믹은 *마찰이 아니라 그 일기장을 산 이유(매력)*. 안 산 사람한테 기자질하면 마찰이지만, 산 사람한텐 아님.
- **구조**: **공통 베이스 메커니즘(고정) + 일기장별 페르소나 레이어(인격·호기심·말투).**

### 결정 (페르소나)
- **(가)** 페르소나 = 일기장 속성. **인격은 다름**(친구/리포터/소설가), **기억은 유저 단위 하나**.
- **(나) 공통 바닥(모든 인격이 상속)**: ① 심문 안 함 ② 추측-확인으로 감상 끌어냄 ③ 사실 날조 금지 ④ "숙제처럼" 안 느껴짐(마찰 제로). 캐릭터가 달라도 이 4개는 동일.
- **(다) 🔴 유일한 실질 리스크 = "쟤가 날 왜 알지?"**(공유 기억 + 다른 인격 → 감시 느낌). → **막는 법: 기억을 인격에 맞게 표면화.** 같은 사실도 친구="응원/내가 기억해", 리포터=**"제보/취재 자료로 들었습니다"**. 기억이 "우리 사이"가 아니라 "취재거리"로 들어오면 uncanny 제거.
- **(라) 경계 = 일기장(책장 IA)**. 유저가 책을 골라 펼치고 그 화자를 기대하며 진입 → 한 화면 안에서 인격 안 바뀜 → 톤 휘플래시 없음.
- **(마) 감정 홈 = 친구(일반)**. A+B 타겟의 매일 드라이버는 친구. 리포터/소설가는 *특별한 기록*용. §3 감정아크("얘가 날 기억한다")의 앵커는 친구가 짊 → 인격 분리로 옅어질 정서적 연속성을 보완.
- **(바) 리포터 = 존댓말**. 말투는 프롬프트 몇 줄(가벼운 결정). 단 존댓말이 "차갑게/거리두기"로 새지 않게 — formats.ts "친근한 기자" 바닥 유지.
- **(미세 이슈, 후순위)**: 같은 날 두 일기장에 같은 사건 쓰면 *두 번 인터뷰*하는 중복. 빈도 낮음(일반=매일/신문=특별한 날) → "다른 일기장 = 같은 사건의 다른 각도"로 두고 중복 감지는 후순위.

### UX 예시 (지우 — 일반+신문 둘 다, 데모데이 날)
- 🌅 **일반(친구·반말)**: "오늘 데모데이 날이지?! 끝나고 얘기해줘." → 저녁 "발표 어땠어?" → 추측확인("박수받을 때 찡했지?") → *담백한 감정 중심 일기.*
- 🌙 **신문(리포터·존댓말)**: "데모데이가 있었다고 **들었습니다**. 취재 좀 하겠습니다 — 몇 시, 어디서?"(육하원칙) → "한 문장으로 표현하면?"(인용) → *헤드라인/리드/인용 기사.*
- 같은 사건, 다른 인격, 같은 기억 — "들었습니다"가 uncanny 차단 장치.

## 3. A. 인터뷰 전략 ✅ 확정

> "인격별 호기심" = **포맷별 수집 체크리스트**(formats.ts `requiredInfo` 계승). 운영 방식이 핵심.

### 포맷별 수집 체크리스트(=호기심) + enough 기준 — 포맷 정의의 일부(agent-v2 §8.2)
| 인격 | 체크리스트 | enough 기준 |
|---|---|---|
| 친구(일반) | 사건/활동 · 시간 흐름 · **감정·생각** · 기억에 남는 순간 | 흐름 + *확인된* 감상 |
| 리포터(신문) | 헤드라인 사건 · 육하원칙 · 배경/원인 · 수치/디테일 · **인용 코멘트** | 헤드라인1 + 육하원칙 대체로 + 인용 |
| 소설가(소설) | 중심 장면 · 감각 디테일 · 인물/관계/대사 · **내면** · 관통 분위기 | 중심장면1 + 감각·감정 풍부 |

### 결정 — Fork A1 = (c) 하이브리드 ✅
PoC(자율)=조기종료 / agent-v2(매 턴 강제 구조화 평가)=심문·지연·경직 → 둘 다 탈락. 채택:
- **한 번의 LLM 호출**이 *자연스러운 답변(인격)* + *구조화 꼬리표*(채워진/건너뛴 항목 · `enough` bool · 다음에 기울일 빈칸)를 같이 출력. 별도 호출 없어 비용/지연 안 늘고 상태는 명시적.
- **체크리스트는 질문을 *지시*하지 않고 *편향*만** — "빈칸 채우려 자연스러움 희생 금지, 유저가 꺼리면 skip 처리 후 마무리"(§2-4 계승, 공통바닥 심문금지와 일관).

### (c)에 따라오는 운영 규칙 ✅
- **하루 누적**: 구조화 상태를 *대화(하루)에 영속* → 재진입 시 로드. greeting(예정기억 훅)은 **첫 진입만**, 재진입은 "이어서"("아까 면접 얘기하다 말았는데, 결과 어땠어?"). (§1-1 해소)
- **enough = 부드러운 제안**: `enough`가 처음 true로 뒤집힐 때만 AI 능동 제안("일기로 남겨줄까?"). 버튼은 상시(s3.1 §5.2-4). 제안 후 계속 떠들면 다시 모으되 **매 턴 재권유 금지(나그 스로틀)**. (§1-2 해소)
- **🔑 감상 충족 판정**: "감정·생각" 항목은 **AI 추측이 아니라 유저가 긍정/수정한 것만** 채워진 것으로 침(추측-확인, §2 공통바닥). 사건만 있고 감상 비면 `enough=false`.
- **음성(STT)**: 긴 답변이 여러 항목을 한 번에 채움 → 이미 채운 건 재질문 안 함(상태 기반 자동). 음성=감정 원재료 풍부 → 소설/일반 유리.
- **사진**: gather 중 `requestPhoto`(PoC 계승, s3.1 §5.2-3).

## 4. C. 생성 가드 ✅ 확정

> 대부분 계승(s3.1 §5.2-5 + agent-v2 §2 + formats.ts). 일기장 모델/음성/보강이 만든 엣지 5개를 추가 확정. *테스트하며 개선.*

### 계승 가드 ✅
| 가드 | 내용 |
|---|---|
| 사실 날조 금지 | 유저가 말/확인한 것만. 감정·순서·외부사실(날씨/뉴스) 지어내기 X |
| 시간순서 단정 금지 | 모호하면 순서 단정 X |
| 길이 = 입력 비례 | 입력 적으면 짧게, 억지 패딩 X |
| 포맷별 톤 | 일반=담백 / 소설=윤색 / 신문=사실·육하원칙·과장금지 |
| 이미지 인라인 | 캡션 근거 위치만, 캡션 원문 베끼기 X |

### 새 엣지 가드 ✅
- **C1 연속성 주입 vs 창작금지**: 기억이 주입돼도 일기엔 **오늘 대화가 실제로 닿은 기억만**. DB에 있어도 오늘 안 나온 건 ✗. 연속성 표현은 *오늘 대화 + 과거 기억 둘 다 실재*일 때만. (§2-다의 생성 버전)
- **C2 소설 "윤색" 경계**: 사실관계(누가/뭘/언제/어디서/감정)는 일반과 동일 가드(날조 금지). **윤색 허용 = 문장·리듬·은유·시점·구조뿐.** 🔑 **감각 디테일도 출처 제한** — 유저가 말했거나 사진/날씨로 주어진 것만. 대화에 없던 감각("창밖 빗소리")은 분위기용이라도 못 지어냄.
- **C3 thin day = 포맷별로 짧게**: 일반→담백 / 소설→짧은 장면 스케치(없는 사건 채우기 X) / 신문→억지 헤드라인 대신 **"단신"**.
- **C4 음성(STT) 원재료**: 정제/압축 OK, **내용 추가 금지**. STT 오타는 문맥 교정 가능하나 사실 추가 X. 말끝·강조 감정 뉘앙스는 살림.
- **C5 보강 재생성**: **누적 transcript로 전체 재생성**(증분 X, 일관성). 수정으로 다듬은 게 있으면 보강이 덮어씀("보강 후 다시 수정"). "수정 문장 보존하며 보강"은 후순위(복잡도↑).

## 5. D. 문체 / 길이 ✅ 확정

> 북극성 *"다시 보고 싶은 **내 문체**의 일기"* 직결. 길이=입력비례(확정) 위에 채움.

- **D1 일기 문체의 주체 = 대화 말투가 아니라 포맷 화자.** 친구(반말·입말)는 인터뷰만, 일기는 다른 목소리:
  - 일반 → **유저 1인칭 글말체**("~했다"). 대화체(반말) ≠ 일기체.
  - 신문 → 기자 3인칭 기사체(유저=주인공). 소설 → 서술자(1/3인칭), 장면+내면.
- **D2 "유저 어휘 에코" = 명시 품질 레버.** 일기가 대화 속 유저 실제 표현·단어를 되살림 → "내 문체". 적용 위치: 일반=본문 전체 / 신문=인용구 원문 / 소설=대사+서술 일부.
- **D3 포맷별 구조·길이 성격**: 일반=도입–전개–여운 흐름(나열 X) / 신문=리드+육하원칙+인용 / 소설=장면 밀도.
- **D4 길이 = soft(입력 비례), hard cap 없음.** 단 일기다운 최소 형태 유지 — thin day여도 "오늘 ~함" 한 줄 X, *흐름*은 있게(C3 단신도 리드+한 단락).
- **D5 (후순위·비전) 문체 학습·성장** — 누적될수록 유저 목소리에 맞춰감 = 기억 위 비전(M3+). 1차는 매 transcript 기반.
